# 06. Технологический стек — Industrial Vibration Analyzer

## Назначение документа

Этот документ описывает, какие технологии, библиотеки и инструменты используются в проекте, почему они были выбраны и какую роль каждый из них играет.

---

## Язык программирования

### Python 3.11+

Python выбран как основной язык по следующим причинам:

- богатейшая экосистема библиотек для научных вычислений (NumPy, SciPy, Pandas) и построения графиков (Matplotlib, Plotly);
- де-факто стандарт в инженерном и научном программировании;
- читаемый синтаксис, позволяющий формулировать алгоритмы близко к их математическому описанию;
- возможность сборки в автономный `.exe` через PyInstaller.

Минимальная версия: Python 3.11. Причина — улучшенная производительность и читаемые сообщения об ошибках (`tomllib` встроен, `match/case` стабилен).

---

## Интерфейс пользователя

### PySide6 (Qt for Python)

PySide6 — официальный Python-биндинг к фреймворку Qt 6. Выбран для построения настольного интерфейса по следующим причинам:

- Qt предоставляет зрелый, производительный и хорошо документированный набор компонентов для создания настольных приложений;
- нативный внешний вид на Windows 11;
- механизм сигналов и слотов обеспечивает слабую связанность между компонентами интерфейса;
- поддержка многопоточности через `QThread` и `QRunnable` позволяет выполнять вычисления в фоне без блокировки интерфейса;
- официальная лицензия LGPL позволяет использование в коммерческих продуктах.

Роль в проекте: вся директория `ui/` построена на PySide6.

### Matplotlib (встроенный в PySide6 через `FigureCanvasQTAgg`)

Matplotlib используется для отрисовки графиков непосредственно внутри виджетов Qt. Это даёт полный контроль над внешним видом графиков и позволяет переиспользовать тот же код построения графиков при генерации PDF-отчётов.

## Научные вычисления

### NumPy

Базовая библиотека для работы с числовыми массивами. Используется во всех расчётных модулях `core/`. Обеспечивает векторизованные операции без Python-циклов, что критично для обработки сигналов длиной в миллионы отсчётов.

### SciPy

Библиотека научных алгоритмов. В проекте используются следующие модули:

- `scipy.signal`: фильтр Баттерворта, вычисление PSD методом Уэлча, поиск пиков;
- `scipy.stats`: статистические характеристики распределения сигнала;
- `scipy.interpolate`: линейная интерполяция при заполнении пропусков.

### Pandas

Используется для:
- чтения CSV и Excel при импорте данных (`infrastructure/readers/`);
- хранения табличных данных в промежуточном представлении;
- экспорта результатов в CSV.

Pandas не используется внутри расчётных модулей `core/` — там работают чистые NumPy-массивы. Это обеспечивает максимальную производительность вычислений.

### Polars (опционально, для больших файлов)

Polars — высокопроизводительная библиотека для работы с табличными данными, написанная на Rust. Включена в зависимости как ускоритель чтения файлов Parquet при объёме данных более 100 МБ. Переключение между Pandas и Polars прозрачно для слоёв `app/` и `core/` — ридеры в `infrastructure/` возвращают единый формат.

### ReportLab

Библиотека для программной генерации PDF-документов. Выбрана за полный контроль над содержимым и оформлением страницы без зависимости от внешних рендереров или LaTeX. Используется в `infrastructure/writers/pdf_report_writer.py`.

Библиотека `pyarrow` используется для чтения и записи файлов Parquet. Это единственная зависимость, добавленная специально для этого формата.

## Сборка и дистрибуция

### PyInstaller

Инструмент для упаковки Python-приложения вместе со всеми зависимостями в автономный `.exe`. После сборки приложение запускается на любой машине с Windows 11 без установленного Python.

Конфигурация PyInstaller хранится в файле `scripts/iva.spec`. Сборка выполняется командой `python scripts/build_installer.py`.

### Inno Setup

Бесплатный инструмент для создания установщика Windows (`.exe`). Создаёт `setup.exe`, который устанавливает приложение в папку `%ProgramFiles%\IVA\`, создаёт ярлык на рабочем столе и запись в «Программах и компонентах» для корректного удаления.

Скрипт Inno Setup хранится в `scripts/installer.iss`.

## Инструменты разработки

### pytest

Фреймворк для написания и запуска автоматических тестов. Все тесты в директории `tests/` выполняются командой `pytest`. Конфигурация в `pyproject.toml`.

### pytest-cov

Измеряет покрытие кода тестами и формирует HTML-отчёт. Цель: не менее 80% покрытия для модулей `core/`.

### black

Автоматический форматировщик кода. Устраняет любые разногласия о стиле форматирования — код всегда выглядит одинаково. Настроен на максимальную длину строки 100 символов.

### ruff

Быстрый линтер на Rust. Проверяет код на типичные ошибки, неиспользуемые импорты, несоответствие именования и другие проблемы. Настройки в `pyproject.toml`.

### mypy

Статическая проверка типов. Аннотации типов обязательны для всех функций в `core/` и `app/`. Это позволяет обнаруживать многие ошибки до запуска кода.

### pre-commit

Система хуков Git, автоматически запускающая `black`, `ruff` и `mypy` перед каждым коммитом. Это гарантирует, что в репозиторий не попадает некорректно отформатированный или содержащий очевидные ошибки код.

## Управление зависимостями

### pip + `requirements.txt`

Зависимости фиксируются в `requirements.txt` с точными версиями. Это гарантирует воспроизводимость сборки на любой машине.

Разделение зависимостей:
- `requirements.txt` — производственные зависимости (то, что войдёт в `.exe`);
- `requirements-dev.txt` — зависимости разработки (pytest, black, ruff, mypy, pre-commit).

### pyproject.toml

Единый файл конфигурации проекта, содержащий: метаданные пакета, настройки black, ruff, mypy, pytest. Соответствует стандарту PEP 517/518.
