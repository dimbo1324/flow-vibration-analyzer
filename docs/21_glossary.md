# 21. Глоссарий — Industrial Vibration Analyzer

## Назначение документа

Этот документ содержит определения терминов, используемых в документации и коде проекта. Цель — устранить двусмысленность: каждое понятие должно означать одно и то же для всех участников проекта.

Термины сгруппированы по областям знаний. Внутри каждой группы — по алфавиту.

## Физика и гидродинамика

**Вихревая дорожка Кармана (Kármán vortex street)** — периодическая структура вихрей, образующихся в следе за обтекаемым телом. Вихри отрываются поочерёдно с противоположных сторон тела, создавая знакочередующиеся поперечные силы. Именно эта нестационарность является источником потоково-индуцированных вибраций.

**Захват частоты, lock-in** — режим течения, при котором частота срыва вихрей синхронизируется с собственной частотой колебаний конструкции. В этом режиме амплитуда вибраций резко возрастает даже при небольшом изменении скорости потока. Наиболее опасный сценарий потоково-индуцированных вибраций.

**Коэффициент демпфирования (ζ, damping ratio)** — безразмерная характеристика затухания колебательной системы. При ζ < 1 система совершает затухающие колебания, при ζ ≥ 1 — апериодически возвращается в положение равновесия. Типичные значения для трубных систем: 0,002–0,05.

**Момент вихреобразования, частота срыва вихрей (fs, vortex shedding frequency)** — частота, с которой вихри отрываются от тела в единицу времени, Гц. Определяется из числа Струхаля: `fs = St · V / D`.

**Номер Рейнольдса, Re (Reynolds number)** — безразмерное число подобия, характеризующее отношение инерционных сил к вязким: `Re = V · D / ν`. Определяет режим течения (ламинарный или турбулентный) и форму вихревой дорожки.

**Номер Струхаля, St (Strouhal number)** — безразмерное число, связывающее частоту срыва вихрей с характеристиками тела и потока: `St = fs · D / V`. Для одиночного цилиндра в развитом турбулентном режиме St ≈ 0,20–0,21.

**Поперечный поток (cross-flow)** — конфигурация обтекания, при которой вектор средней скорости потока перпендикулярен оси цилиндра. Наиболее распространённая конфигурация в трубных пучках теплообменников.

**Потоково-индуцированные вибрации (Flow-Induced Vibrations, FIV)** — вибрации конструктивных элементов, вызванные нестационарными гидродинамическими силами при обтекании их потоком жидкости или газа. Включают: вихревые вибрации (vortex-induced vibrations), флаттер, акустический резонанс.

**Приведённая скорость (Vr, reduced velocity, velocity ratio)** — безразмерный параметр `Vr = V / (fn · D)`, характеризующий близость рабочего режима к зоне lock-in. Зона повышенного риска для одиночного цилиндра: Vr ≈ 4–8.

**Собственная частота (fn, natural frequency)** — частота свободных колебаний конструктивного элемента при отсутствии демпфирования, Гц. Определяется геометрией, материалом и граничными условиями закрепления.

**Тандем цилиндров (tandem cylinders)** — конфигурация из двух цилиндров, расположенных последовательно в направлении потока. Аэродинамические и вибрационные характеристики тандема существенно отличаются от одиночного цилиндра из-за взаимодействия следов.

## Обработка сигналов

**Децибел (dB)** — логарифмическая единица отношения мощностей: `dB = 10 · log₁₀(P₂/P₁)`. В контексте PSD: `PSD_dB = 10 · log₁₀(PSD)`. Логарифмическая шкала позволяет одновременно видеть на графике пики разной амплитуды.

**Оконная функция (window function)** — весовая функция, применяемая к сегменту сигнала перед вычислением FFT для снижения утечки спектра. Приложение поддерживает: Ханнинга (по умолчанию), Хэмминга, прямоугольное окно.

**Постоянная составляющая (DC component)** — нулевая частотная компонента сигнала, соответствующая его среднему значению. В спектре проявляется как пик на частоте 0 Гц. Удаляется на первом шаге предобработки.

**Пик-фактор (crest factor)** — отношение пикового значения сигнала к его RMS. Высокий пик-фактор (> 4) указывает на наличие ударных нагрузок или выбросов.

**Разрешение по частоте (frequency resolution, Δf)** — наименьшее расстояние между соседними частотными точками спектра: `Δf = fs / nperseg`. Определяет, насколько близкорасположенные пики можно различить.

**RMS (Root Mean Square, среднеквадратичное значение)** — квадратный корень из среднего квадрата значений сигнала. Характеризует эффективную амплитуду и связан с энергией колебаний. Для синусоиды с амплитудой A: `RMS = A / √2`.

**Спектральная плотность мощности (PSD, Power Spectral Density)** — функция, описывающая распределение мощности сигнала по частотам. Единица: [физ. ед.]²/Гц. Интеграл PSD по частоте равен дисперсии сигнала.

**Утечка спектра (spectral leakage)** — артефакт FFT, при котором мощность одного пика «растекается» по соседним частотным точкам из-за конечной длины сегмента. Подавляется применением оконных функций.

**Фильтр Баттерворта (Butterworth filter)** — цифровой фильтр с максимально плоской амплитудно-частотной характеристикой в полосе пропускания (без пульсаций). Выбран в приложении за отсутствие амплитудных искажений в полосе пропускания.

**Частота дискретизации (sampling rate, fs)** — количество отсчётов в секунду, Гц. Максимальная частота, которую можно воспроизвести: `fs / 2` (теорема Найквиста).

**Частота Найквиста (Nyquist frequency)** — максимальная частота, которая может быть однозначно представлена при данной частоте дискретизации: `fN = fs / 2`. Спектральный анализ ограничен этой частотой.

**Метод Уэлча (Welch's method)** — метод оценки спектральной плотности мощности путём усреднения периодограмм перекрывающихся сегментов сигнала с применением оконной функции. Снижает дисперсию оценки PSD по сравнению с прямым FFT.

## Архитектура приложения

**Инвариант (invariant)** — условие, которое всегда выполняется для объекта данного класса. Инварианты описаны в моделях данных (`10_data_models_and_schemas.md`) и проверяются в конструкторе или методе валидации.

**Конвейер обработки (processing pipeline)** — последовательность шагов преобразования данных, где выход каждого шага является входом следующего. Полная схема описана в `09_processing_pipeline.md`.

**Модель данных (data model)** — структура, описывающая состав, типы и ограничения данных. В проекте реализована как Python `@dataclass`. Все модели хранятся в `core/models/`.

**Сигнал Qt (Qt signal)** — механизм оповещения в PySide6: объект испускает сигнал при наступлении события, все подключённые к нему слоты (обработчики) вызываются автоматически. Обеспечивает слабую связанность компонентов.

**Слой (layer)** — группа модулей, объединённых общей ответственностью. Проект имеет четыре слоя: `ui` (представление), `app` (координация), `core` (предметная область), `infrastructure` (I/O и внешние форматы).

**Слот Qt (Qt slot)** — функция-обработчик, подключённая к сигналу Qt. Вызывается автоматически при испускании сигнала.

**Фикстура теста (test fixture)** — заранее подготовленное состояние или данные, используемые в тесте. В pytest реализуется через декоратор `@pytest.fixture`. Хранится в `tests/fixtures/`.

## Инструменты и форматы

**black** — автоматический форматировщик кода Python. Принимает однозначные решения о форматировании, устраняя споры о стиле.

**MAD (Median Absolute Deviation)** — устойчивая к выбросам оценка разброса данных: `MAD = median(|xi − median(x)|)`. Используется в алгоритме обнаружения выбросов вместо стандартного отклонения.

**MAE (Mean Absolute Error)** — средняя абсолютная погрешность между двумя наборами данных.

**MAPE (Mean Absolute Percentage Error)** — средняя абсолютная процентная погрешность. Не применяется при нулевых значениях в знаменателе.

**mypy** — инструмент статической проверки типов для Python. Обнаруживает несоответствия аннотаций типов без запуска кода.

**Parquet** — бинарный колоночный формат хранения данных Apache Parquet. Обеспечивает высокую скорость чтения и эффективное сжатие для числовых таблиц.

**pip-audit** — инструмент проверки зависимостей Python на известные уязвимости из базы данных CVE.

**pre-commit** — система Git-хуков, выполняющая автоматические проверки кода перед каждым коммитом.

**RMSE (Root Mean Square Error)** — среднеквадратичное отклонение между экспериментальными и расчётными данными. Особенно чувствителен к большим отклонениям.

**ruff** — высокоскоростной линтер Python, реализованный на Rust. Проверяет стиль, неиспользуемые импорты, нарушения соглашений об именовании.

**PyInstaller** — инструмент упаковки Python-приложения в автономный исполняемый файл (`.exe` на Windows).
