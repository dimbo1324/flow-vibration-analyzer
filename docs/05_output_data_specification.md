# 05. Спецификация выходных данных — Industrial Vibration Analyzer

## Назначение документа

Этот документ описывает все данные, которые приложение создаёт в результате работы: форматы файлов, структуру отчётов, состав таблиц и метаданных. Пользователь должен точно понимать, что он получит на выходе и в каком виде.

---

## Виды выходных данных

Приложение формирует три вида выходных данных:

**Интерактивный результат** — отображается в интерфейсе немедленно после завершения расчёта. Пользователь может изучать графики, переключаться между вкладками, менять параметры и запускать пересчёт.

**Экспорт в CSV** — структурированная таблица с числовыми результатами, пригодная для последующей обработки в других инструментах.

**Отчёт в PDF** — оформленный документ, содержащий графики, таблицы, физические коэффициенты и итоговое заключение. Предназначен для передачи коллегам, заказчику или включения в техническую документацию.

---

## PDF-отчёт

### Структура отчёта

PDF-отчёт состоит из следующих разделов:

**Титульная страница.** Название анализа, имя файла с данными, дата и время формирования отчёта, версия приложения.

**Раздел 1 — Параметры анализа.** Таблица с перечнем всех входных параметров, которые пользователь задал перед запуском анализа: геометрия, параметры среды, настройки фильтрации и спектрального анализа.

**Раздел 2 — Качество данных.** Количество строк, доля пропусков, количество удалённых выбросов, обнаруженные предупреждения.

**Раздел 3 — Временна́я область.** График исходного и отфильтрованного сигнала. Таблица с основными статистическими характеристиками: минимум, максимум, среднее, RMS, пик-фактор.

**Раздел 4 — Спектральный анализ.** График спектральной плотности мощности с отмеченными пиками. Таблица найденных пиков с колонками: частота (Гц), амплитуда (единицы сигнала²/Гц), ширина на уровне -3 дБ (Гц), интерпретация.

**Раздел 5 — Физические коэффициенты.** Таблица с рассчитанными значениями: Re, St, расчётная частота срыва вихрей, приведённая скорость, уровень риска.

**Раздел 6 — Сравнение с расчётом (если применимо).** Совмещённые графики экспериментального и расчётного профилей. Таблица метрик качества: RMSE, MAE, MAPE, коэффициент корреляции.

**Раздел 7 — Итоговое заключение.** Уровень риска резонанса, краткое текстовое заключение и рекомендация инженеру.

**Приложение.** Полная таблица параметров, журнал расчёта с временны́ми метками.

### Формат и оформление PDF

- размер страницы: A4;
- шрифт основного текста: 11 pt, sans-serif;
- шрифт для таблиц и кода: 9 pt, monospace;
- графики сохраняются в разрешении 150 dpi в векторном виде, если это позволяет выбранная библиотека;
- цветовая схема: тёмный фон для графиков в приложении заменяется светлым при экспорте в PDF для удобства печати.

## Экспорт в CSV

### Файл результатов спектрального анализа (`spectrum_results.csv`)

| Колонка | Тип | Описание |
|---|---|---|
| `frequency_hz` | float | Частота, Гц |
| `psd_value` | float | Спектральная плотность мощности |
| `is_peak` | bool | Признак обнаруженного пика |
| `peak_interpretation` | string | Интерпретация пика (или пусто) |

### Файл временно́го сигнала (`signal_processed.csv`)

| Колонка | Тип | Описание |
|---|---|---|
| `time_s` | float | Время, с |
| `signal_raw` | float | Исходный сигнал |
| `signal_filtered` | float | Сигнал после фильтрации |
| `rms_window` | float | Скользящий RMS |

### Файл физических коэффициентов (`physics_summary.csv`)

| Колонка | Тип | Описание |
|---|---|---|
| `parameter_name` | string | Название параметра |
| `value` | float | Числовое значение |
| `unit` | string | Единица измерения |
| `description` | string | Краткое описание |

### Файл сравнения (`validation_results.csv`)

| Колонка | Тип | Описание |
|---|---|---|
| `coordinate` | float | Координата (позиция по профилю) |
| `experiment_value` | float | Экспериментальное значение |
| `cfd_value` | float | Расчётное значение |
| `absolute_error` | float | Абсолютное отклонение |
| `relative_error_percent` | float | Относительное отклонение, % |

## Метаданные в выходных файлах

Все выходные файлы содержат блок метаданных в заголовке (для CSV — в виде строк комментариев с символом `#`; для PDF — на титульной странице).

Состав метаданных:
- версия приложения IVA;
- дата и время формирования файла;
- имя исходного входного файла;
- хэш-сумма MD5 исходного файла (для воспроизводимости);
- параметры анализа в компактной записи.

Наличие хэш-суммы позволяет инженеру через любое время убедиться, что отчёт был построен именно по тому файлу данных, который он имеет в наличии.

---

## Правила именования выходных файлов

Имена файлов формируются автоматически по шаблону:

```
{исходное_имя_файла}_{тип_экспорта}_{дата}_{время}.{расширение}
```

Пример:
```
measurement_run014_spectrum_results_2025-06-10_12-01-14.csv
measurement_run014_report_2025-06-10_12-01-14.pdf
```

Пользователь может изменить путь и имя файла в диалоге сохранения, но шаблон по умолчанию обеспечивает уникальность и прослеживаемость.

---

## Папка для хранения результатов

По умолчанию все выходные файлы сохраняются в папке:

```
%USERPROFILE%\Documents\IVA\results\
```

Пользователь может изменить путь в настройках приложения. Папка создаётся автоматически при первом запуске.
