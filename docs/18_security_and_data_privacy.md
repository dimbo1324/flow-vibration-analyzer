# 18. Безопасность и конфиденциальность данных — Industrial Vibration Analyzer

## Назначение документа

Этот документ описывает меры безопасности, принятые в проекте, и принципы обращения с данными пользователя. Безопасность рассматривается как часть архитектуры, а не дополнение к ней.

---

## Модель угроз

IVA — настольное приложение без сетевого компонента. Оно не передаёт данные через интернет, не имеет сервера и не использует облачные сервисы. Поверхность атаки ограничена двумя источниками.

**Входные файлы.** Пользователь загружает файлы из внешних источников. Вредоносный файл может попытаться вызвать переполнение буфера при чтении, выполнить произвольный код через уязвимости в библиотеках парсинга или привести к некорректным результатам, которые будут приняты за корректные.

**Зависимости.** Сторонние библиотеки (NumPy, SciPy, Pandas, PyArrow) могут содержать уязвимости. Необновлённые зависимости — наиболее распространённый вектор атак на десктопные приложения.

---

## Принципы безопасности

### Нет секретных данных в коде

Приложение не использует API-ключи, пароли, токены или иные секреты. Функционал не требует авторизации. Если в будущем потребуется хранение конфиденциальных данных, для этого будет использоваться `keyring` (Windows Credential Manager), а не файлы конфигурации.

### Проверка входных данных

Каждый входной файл проходит структурную проверку до обработки. Приложение не доверяет содержимому файла автоматически:

- тип файла определяется по содержимому (magic bytes), а не только по расширению;
- размер файла проверяется до чтения;
- числовые значения проверяются на допустимые диапазоны перед использованием в формулах.

Проверка выполняется в `core/models/signal_data.py` и `infrastructure/readers/`.

### Актуальные зависимости

При каждой сборке релиза зависимости проверяются на известные уязвимости:

```bash
pip-audit --requirement requirements.txt
```

Зависимости с уязвимостями класса «высокий» или «критический» обновляются до выпуска релиза. Зависимости с уязвимостями класса «средний» фиксируются в `20_roadmap.md` и обновляются в ближайшем PATCH-релизе.

## Конфиденциальность данных

### Данные остаются на компьютере пользователя

Приложение работает полностью офлайн. Никакие входные файлы, результаты, параметры или журналы не покидают компьютер пользователя. Отсутствие сетевых запросов является архитектурным решением, а не случайностью.

### Что хранится локально

| Тип данных | Расположение | Кто создаёт |
|---|---|---|
| Настройки приложения | `%APPDATA%\IVA\settings.toml` | Приложение |
| Файлы журналов | `%USERPROFILE%\Documents\IVA\logs\` | Приложение |
| Результаты анализа | `%USERPROFILE%\Documents\IVA\results\` | Пользователь (по запросу) |

Входные файлы пользователя не копируются приложением. Они читаются по пути, указанному пользователем, и остаются на месте.

### Состав файлов журнала

Журналы содержат: временны́е метки, имена файлов без полного пути, агрегированные характеристики данных (количество строк, доля пропусков), числовые результаты расчётов. Журналы не содержат числовых данных из загруженных файлов.

---

## Безопасная работа с файлами

При чтении файлов применяются следующие ограничения:

- максимальный размер файла для автоматической обработки: 500 МБ (файл большего размера требует явного подтверждения);
- при чтении Excel-файлов отключены макросы и внешние ссылки (`openpyxl` используется в режиме `read_only=True`);
- при чтении CSV не выполняется интерпретация формул (значения, начинающиеся с `=`, `+`, `-`, `@`, принимаются как строки).

---

## Настройки приложения

Файл настроек `settings.toml` содержит только пользовательские предпочтения интерфейса (путь для сохранения результатов, последний использованный путь к файлу, параметры анализа по умолчанию). Он не содержит данных, относящихся к конкретным анализам.

Файл настроек не шифруется, поскольку не содержит конфиденциальных данных. Если в будущей версии появятся чувствительные настройки — они будут перенесены в Windows Credential Manager.
