# 04. Спецификация входных данных — Industrial Vibration Analyzer

## Назначение документа

Этот документ описывает, какие данные принимает приложение на входе: форматы файлов, обязательные и необязательные поля, допустимые диапазоны значений, правила проверки и типичные сценарии получения данных.

---

## Источники входных данных

Приложение работает с данными, полученными двумя основными способами.

**Данные физических измерений** — записи с датчиков, установленных на испытательном стенде или в промышленном оборудовании: акселерометры, пьезоэлектрические датчики давления, лазерные доплеровские анемометры (LDA/LDV), системы PIV. Данные, как правило, экспортируются из программного обеспечения измерительных систем в виде таблиц с равномерной временной разбивкой.

**Данные CFD-расчётов** — профили скорости, поля давления и пульсации, полученные в результате численного моделирования в ANSYS CFX, OpenFOAM, ЛОГОС или аналогичных пакетах. Эти данные используются во вкладке сравнения для оценки точности расчётной модели.

---

## Поддерживаемые форматы файлов

### CSV (Comma-Separated Values)

Наиболее распространённый формат для экспорта из измерительных систем. Приложение поддерживает CSV-файлы со следующими характеристиками:

- разделитель: запятая, точка с запятой или табуляция (определяется автоматически);
- десятичный разделитель: точка или запятая (определяется автоматически по первым 100 строкам);
- кодировка: UTF-8, Windows-1251 (CP1251), Latin-1 (определяется автоматически);
- первая строка может содержать заголовки колонок (определяется автоматически);
- комментарии в начале файла, начинающиеся с символа `#`, пропускаются.

### Parquet

Бинарный колоночный формат Apache Parquet. Рекомендован для больших объёмов данных (более 10 млн строк) благодаря высокой скорости чтения и эффективному сжатию. Используется при экспорте из Python-скриптов или систем сбора данных промышленного уровня.

### Excel (`.xlsx`)

Формат Microsoft Excel. Поддерживается для совместимости с лабораторными системами, которые экспортируют данные только в этом формате. При наличии нескольких листов пользователь выбирает нужный лист через диалог.

## Структура данных временного ряда

Входной файл должен содержать как минимум две колонки: временну́ю ось и один измеренный сигнал. Остальные колонки необязательны.

### Обязательные колонки

**Временна́я ось.** Колонка с монотонно возрастающими значениями времени. Допустимые единицы: секунды, миллисекунды (определяются пользователем). Интервал между отсчётами должен быть постоянным. Допускается отклонение не более 0,01% от номинального интервала — это соответствует погрешности типовых систем сбора данных.

**Хотя бы один измеренный сигнал.** Любая числовая колонка, которую пользователь назначит на роль анализируемого сигнала.

### Необязательные колонки и их роли

Пользователь назначает колонки на следующие роли через интерфейс:

| Роль | Описание | Типичное название колонки |
|---|---|---|
| Время | Временна́я ось | `time`, `t`, `Time [s]` |
| Виброускорение X | Ускорение по оси X | `acc_x`, `acceleration_x` |
| Виброускорение Y | Ускорение по оси Y | `acc_y`, `acceleration_y` |
| Виброускорение Z | Ускорение по оси Z | `acc_z`, `acceleration_z` |
| Давление | Пульсации давления | `pressure`, `p`, `dp` |
| Скорость потока | Компонента скорости | `velocity`, `v_x`, `u` |
| Расход | Объёмный расход | `flow_rate`, `Q` |

Для анализа требуется назначение хотя бы одной роли помимо временно́й оси. Одновременно можно анализировать несколько сигналов.

## Параметры физической установки

Помимо файла с данными, пользователь вводит параметры геометрии и свойств среды. Эти параметры используются для расчёта физических коэффициентов и оценки риска резонанса.

### Геометрические параметры

| Параметр | Обозначение | Единица | Допустимый диапазон | Обязательный |
|---|---|---|---|---|
| Наружный диаметр цилиндра | D | мм | 1–500 | Да |
| Расстояние между центрами цилиндров (для тандема) | L | мм | D–20·D | Нет |
| Собственная частота конструкции | fn | Гц | 1–10 000 | Нет |
| Коэффициент демпфирования | ζ | — | 0.001–0.2 | Нет |

### Параметры течения и среды

| Параметр | Обозначение | Единица | Допустимый диапазон | Обязательный |
|---|---|---|---|---|
| Средняя скорость потока | V | м/с | 0.01–100 | Да |
| Плотность среды | ρ | кг/м³ | 0.5–15 000 | Да |
| Динамическая вязкость | μ | Па·с | 1e-6–1e-1 | Да |
| Коэффициент преобразования датчика виброускорения | k_acc | (м/с²)/ед | > 0 | Нет |
| Коэффициент преобразования датчика давления | k_press | Па/ед | > 0 | Нет |

Если коэффициент преобразования не указан, сигнал анализируется в условных единицах без физической размерности. Это допустимо для спектрального анализа, но физический смысл RMS-значений в этом случае не определён.

---

## Параметры дискретизации

| Параметр | Описание | Допустимый диапазон |
|---|---|---|
| Частота дискретизации | Количество отсчётов в секунду | 100–100 000 Гц |
| Длительность записи | Общее время наблюдения | не менее 5 секунд |
| Количество отсчётов | Произведение двух предыдущих | не менее 500 |

Если частота дискретизации не может быть определена из файла, пользователь вводит её вручную. Приложение предлагает оценку на основании медианного интервала между соседними отсчётами.

## Правила проверки входных данных

Все входные данные проверяются при загрузке. Ошибки делятся на три категории.

**Критические ошибки** — файл не может быть обработан:
- файл не является таблицей или имеет нераспознанную структуру;
- временна́я ось содержит нечисловые значения;
- временна́я ось немонотонна (есть убывающие интервалы);
- все значения в выбранном сигнале равны нулю или постоянны.

**Предупреждения** — файл загружается, но пользователь информируется о проблемах:
- доля пропущенных значений превышает 5%;
- доля выбросов (по методу скользящего медианного порога) превышает 1%;
- отклонение интервала дискретизации превышает 0,1%;
- диапазон значений сигнала подозрительно мал (возможно, датчик не был подключён).

**Информационные сообщения** — данные корректны, но есть особенности:
- файл содержит более 10 миллионов строк (рекомендуется использовать Parquet);
- обнаружены заголовки на русском языке (автоматически транслитерированы).

## Синтетические данные для тестирования

В комплект поставки входит скрипт `scripts/generate_synthetic_data.py`, который генерирует тестовые файлы с заданными параметрами: частотой дискретизации, длиной записи, частотой срыва вихрей, уровнем шума, наличием выбросов и пропусков.

Синтетические данные используются для автоматического тестирования расчётных модулей и как пример при освоении приложения.
