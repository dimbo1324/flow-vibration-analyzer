# 08. Спецификация интерфейса и пользовательского опыта — Industrial Vibration Analyzer

## Назначение документа

Этот документ описывает принципы построения интерфейса, структуру навигации, поведение ключевых экранов и правила взаимодействия с пользователем. Он служит ориентиром при реализации слоя представления (`ui/`).

---

## Принципы проектирования интерфейса

### Инженерный стиль, а не потребительский

Интерфейс ориентирован на инженера, а не на массового пользователя. Это означает: плотная компоновка информации на экране, минималистичная визуальная иерархия, тёмная цветовая схема (удобна при длительной работе с экраном и при анализе графиков), технические данные представлены числами, а не иллюстрациями.

Ориентир по стилю: промышленный измерительный и аналитический софт (LabVIEW, MATLAB, Ansys Mechanical GUI).

### Рабочий процесс, а не хаотичные инструменты

Навигация построена по этапам работы (01 → 07), а не по типам функций. Пользователь всегда понимает, на каком шаге он находится и что будет следующим.

### Обратная связь на каждое действие

После любого действия пользователь получает отклик: либо немедленный результат, либо индикатор прогресса, либо сообщение об ошибке. Состояние «непонятно, что происходит» недопустимо.

### Предотвращение ошибок, а не наказание за них

Поля ввода имеют подсказки о допустимых диапазонах. Кнопка «Запустить анализ» неактивна, пока не выполнены обязательные условия (загружен файл, назначены колонки). Деструктивные действия требуют подтверждения.

## Цветовая схема

Приложение использует тёмную тему как основную. Светлая тема не предусмотрена в первой версии, но архитектура стилей должна допускать её добавление без переработки кода.

Токены цветов хранятся в `ui/styles/theme.py`. Все компоненты обращаются к цветам только через токены, никогда через прямые значения RGB.

| Токен | Значение | Назначение |
|---|---|---|
| `COLOR_BG` | `#030405` | Фон приложения |
| `COLOR_SURFACE` | `#070809` | Фон окна и панелей |
| `COLOR_PANEL` | `#0b0d10` | Фон карточек |
| `COLOR_BORDER` | `#222832` | Границы элементов |
| `COLOR_TEXT` | `#e7e9ee` | Основной текст |
| `COLOR_MUTED` | `#8b94a3` | Вторичный текст, метки |
| `COLOR_DIM` | `#535d6b` | Неактивные элементы |
| `COLOR_ACCENT` | `#9db7ff` | Акцентные элементы, активные пункты |

| `COLOR_WARN` | `#d4b66e` | Предупреждение |
| `COLOR_BAD` | `#d27b7b` | Критический уровень, ошибка |

## Структура главного окна

Главное окно состоит из четырёх областей:

```
┌─────────────────────────────────────────────────────┐
│  Заголовок окна (title bar)                         │
├─────────────────────────────────────────────────────┤
│  Панель инструментов (toolbar)                      │
├──────────────┬──────────────────────┬───────────────┤
│              │                      │               │
│   Навигация  │   Рабочая область    │   Инспектор   │
│   (sidebar)  │   (main content)     │  (inspector)  │
│              │                      │               │
└──────────────┴──────────────────────┴───────────────┘
```

**Заголовок окна** содержит название приложения, имя текущего файла и индикатор состояния системы.

**Панель инструментов** содержит: кнопку «Импорт», кнопку «Запустить анализ», кнопку «Экспорт отчёта», кнопки управления видимостью боковых панелей.

**Навигация** — список из 7 пунктов, соответствующих этапам рабочего процесса. Активный пункт выделен. Каждый пункт имеет порядковый номер, название и краткое описание.

**Рабочая область** — основное пространство для отображения данных и результатов. Меняется при переключении пунктов навигации.

**Инспектор** — правая панель с текущим результатом анализа, журналом событий и дополнительными параметрами. По умолчанию скрыта, вызывается кнопкой.

## Страницы рабочего процесса

### Страница 01 — Сводка

Главный экран для принятия решения. Отображает четыре ключевых метрики: доминирующий пик, RMS, расчётная частота срыва вихрей, уровень риска.

Ниже метрик — два блока: сводный график сигнала и спектра слева; текстовая интерпретация с таблицей расчётных соотношений справа.

На этой странице инженер должен за 10 секунд понять, безопасен ли текущий режим.

### Страница 02 — Импорт

Зона перетаскивания файла (drag-and-drop). Под ней — таблица обнаруженных колонок с типами данных и оценкой качества. Ещё ниже — форма назначения ролей для колонок.

Пока файл не загружен и роли не назначены — остальные страницы недоступны (затемнены в навигации).

### Страница 03 — Сигнал

Два графика рядом: исходный vs. фильтрованный сигнал; RMS по скользящему окну. Панель настроек фильтрации размещена над графиками. При изменении настроек фильтра графики обновляются.

### Страница 04 — Спектр

Основной график PSD с выделенными пиками. Справа — таблица найденных пиков. Параметры метода Уэлча настраиваются в боковой форме.

### Страница 05 — Физика

Четыре метрики: Re, St, fn, зона lock-in. Под ними — тепловая карта риска по пространству параметров (скорость потока × собственная частота). Форма ввода физических параметров расположена слева от карты.

### Страница 06 — Профили

Два графика: профиль скорости (эксперимент vs. расчёт); векторное поле (предпросмотр). Кнопка загрузки второго файла (расчётных данных) расположена в заголовке страницы.

### Страница 07 — Отчёт

Предпросмотр итогового отчёта на светлом фоне (имитация листа). Кнопки: «Сохранить PDF», «Экспорт CSV». Чекбоксы для включения/исключения отдельных разделов из отчёта.

## Поведение при вычислениях

При нажатии «Запустить анализ»:
1. Кнопка становится неактивной и меняет текст на «Выполняется...».
2. В строке состояния появляется прогресс-бар.
3. Вычисление выполняется в фоновом потоке.
4. После завершения обновляются все страницы, кнопка возвращается в активное состояние.
5. В журнале инспектора добавляется запись с временно́й меткой и результатами.

Если вычисление занимает более 30 секунд, появляется кнопка «Отменить».

## Обработка ошибок в интерфейсе

Ошибки отображаются в виде непрерывного баннера в нижней части рабочей области — не как модальные диалоги, блокирующие работу. Исключение: критические ошибки, требующие явного подтверждения (например, попытка перезаписать существующий отчёт).

Цвет баннера: `COLOR_BAD`. Текст: понятная формулировка без технических деталей. Ссылка «Подробнее» открывает запись в журнале с технической информацией.

## Адаптивность

При ширине окна менее 1200 пикселей:
- правый инспектор скрывается автоматически;
- блоки из двух колонок перестраиваются в одну;
- метрики перестраиваются из 4 в 2 колонки.

При высоте окна менее 720 пикселей:
- размер шрифта метрик уменьшается;
- высота заголовков панелей уменьшается.

## Горячие клавиши

| Клавиша | Действие |
|---|---|
| `Ctrl+O` | Открыть файл |
| `F5` | Запустить анализ |
| `Ctrl+S` | Сохранить отчёт |
| `F` | Переключить фокус-режим |
| `L` | Свернуть/развернуть навигацию |
| `R` | Показать/скрыть инспектор |
| `Esc` | Закрыть развёрнутый график или выйти из фокус-режима |

---

## Доступность

Все интерактивные элементы доступны с клавиатуры. Порядок обхода Tab логически совпадает с визуальным порядком элементов. Кнопки имеют явные текстовые подписи (не только иконки). Иконки-кнопки сопровождаются атрибутом `toolTip`.
