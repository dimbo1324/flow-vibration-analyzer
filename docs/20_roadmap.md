# 20. Дорожная карта — Industrial Vibration Analyzer

## Назначение документа

Этот документ описывает текущее состояние проекта, известные ограничения версии 1.0 и планы по развитию. Он позволяет новому разработчику сразу понять, что уже реализовано, что сознательно отложено и в каком направлении движется проект.

---

## Версия 1.0 — Базовая функциональность (текущий релиз)

### Реализовано

- Импорт файлов CSV, Parquet, Excel с автоматическим определением структуры
- Предобработка сигнала: устранение постоянной составляющей, удаление выбросов, заполнение пропусков, полосовой фильтр Баттерворта
- Спектральный анализ методом Уэлча: окна Ханнинга, Хэмминга, прямоугольное; поиск и интерпретация пиков
- Расчёт RMS: общий, в полосе частот, скользящий тренд
- Физические расчёты: Re, St, частота срыва вихрей, приведённая скорость
- Трёхуровневая оценка риска резонанса с текстовой рекомендацией
- Сравнение с CFD: загрузка расчётного профиля, RMSE, MAE, MAPE, r
- Экспорт: PDF-отчёт, CSV с результатами
- Интерфейс: splitter-компоновка инженерной рабочей станции, компактная навигация,
  инспектор, фокус-режим графиков, индикатор прогресса и согласованные состояния страниц
- Встроенный демо-режим и быстрый старт с шестью синтетическими сценариями
- Установщик `setup.exe` для Windows 11
- Документация: 23 документа

## Известные ограничения версии 1.0

Следующие ограничения приняты сознательно в пользу скорости первого релиза. Они зафиксированы здесь — не спрятаны в коде.

**Нет пакетной обработки.** Анализ нескольких файлов выполняется вручную последовательно. Пользователь должен отдельно открыть каждый файл и сохранить каждый отчёт. Это неудобно при серийных испытаниях.

**Нет редактора временно́го диапазона.** Пользователь не может выбрать для анализа только часть загруженной записи через визуальный инструмент. Ограничение можно обойти, предварительно обрезав файл.

**Светлая тема интерфейса не поддерживается.** PDF-отчёт использует светлый фон, интерфейс — только тёмный. Переключатель темы отсутствует.

**Модель трубных пучков не реализована.** Расчёт выполняется только для одиночного цилиндра и тандема двух цилиндров. Треугольная и квадратная расстановки трубных пучков требуют отдельных таблиц St и не включены в версию 1.0.

**Нет истории сеансов.** Список предыдущих анализов не сохраняется. Пользователь должен помнить, какие файлы он анализировал и где сохранены результаты.

**Отсутствует трёхмерная визуализация.** Вектор поля из PIV-данных отображается только в двумерном предпросмотре.

## Версия 1.1 — Удобство работы

**Планируется:**

Редактор временно́го диапазона на странице «Сигнал». Пользователь выделяет участок записи мышью, и анализ выполняется только по нему. Это позволяет исключить нестационарные переходные участки.

История сеансов в боковой панели. Список 10 последних анализов с датой, именем файла и итоговым уровнем риска. Быстрый переход к сохранённому результату.

Светлая тема интерфейса. Переключатель в настройках с сохранением предпочтения.

## Версия 1.2 — Расширение физических моделей

**Планируется:**

Поддержка прямоугольной и треугольной расстановок трубных пучков. Таблицы St для различных конфигураций и значений шагового отношения.

Модель ASME BPVC Section III Appendix N для оценки риска флаттера и резонанса. Это позволит использовать приложение в рамках нормативных расчётов для атомной энергетики.

Кинематическая вязкость в зависимости от температуры. Встроенная база теплофизических свойств воды, пара и некоторых теплоносителей из нержавеющей стали.

## Версия 2.0 — Пакетный анализ

**Планируется:**

Пакетная обработка: загрузка папки с файлами, применение одной конфигурации ко всем файлам, сводный отчёт по серии испытаний.

Построение зависимости fs(V): автоматическое формирование графика частота–скорость по серии файлов с разными скоростями потока.

Экспорт в Excel-таблицу сводных результатов для серии испытаний.

## Версия 2.1 — Потоковые данные (исследовательская задача)

**Рассматривается:**

Режим мониторинга в реальном времени: подключение к COM-порту или TCP-сокету, непрерывное чтение данных и обновление спектрального анализа с заданной периодичностью. Это потребует переработки архитектуры ридеров и добавления буферизации потоковых данных.

## После релиза 1.0 — известные ограничения (технический долг)

Следующие задачи не вошли в версию 1.0 и задокументированы как известные ограничения:

- **Динамическая децимация при зуме** (Stage 9 backlog): при приближении на графике
  сигнала с > 10 000 точек децимация для видимой области не пересчитывается — отображается
  общая децимированная версия. Запланировано на 1.1.
- **Журнал расчёта в PDF** (Stage 9 backlog): раздел «Приложение: журнал расчёта» в
  PDF-отчёте не включён. Запланировано на 1.1.
- **Разрешение графиков в PDF**: графики встроены в отчёт как векторные фигуры; явная
  настройка 150 dpi как растровый вариант не реализована.
- **Автоматическое перечитывание файла при открытии .vibproj**: при открытии сохранённой
  сессии файл перечитывается, но анализ не запускается автоматически. Пользователь
  должен нажать «Запустить анализ» вручную. Запланировано на 1.1.
