## Фоновое выполнение вычислений

Интерфейс никогда не блокируется во время расчётов. Всё выполнение пайплайна (шаги 1–7 из `09_processing_pipeline.md`) происходит в отдельном потоке через `QThread`.

Архитектура взаимодействия:

```
Главный поток (GUI)          Рабочий поток (QThread)
        │                              │
  [Нажата кнопка]                      │
        │──── запустить анализ ────────►│
        │                       [Запуск пайплайна]
  [Показать прогресс]                  │
        │◄─── прогресс 25% ────────────│
  [Обновить прогресс-бар]              │
        │◄─── прогресс 50% ────────────│
        │◄─── прогресс 75% ────────────│
        │◄─── завершено ───────────────│
  [Обновить все страницы]              │
```

Обмен данными между потоками происходит только через сигналы Qt. Прямое обращение к GUI-объектам из рабочего потока запрещено.

## Векторизованные вычисления

Все числовые расчёты в `core/` используют NumPy-операции над массивами. Python-циклы по элементам массива запрещены в расчётных модулях.

Примеры правильного подхода:

```python
# ПРАВИЛЬНО: векторизованная операция
signal_centered = signal - np.mean(signal)

# НЕПРАВИЛЬНО: Python-цикл
for i in range(len(signal)):
    signal[i] -= np.mean(signal)
```

```python
# ПРАВИЛЬНО: numpy для RMS
rms = np.sqrt(np.mean(signal ** 2))

# НЕПРАВИЛЬНО: Python sum
rms = (sum(x**2 for x in signal) / len(signal)) ** 0.5
```

## Ограничение памяти

Для файлов объёмом более 100 МБ данные читаются и хранятся в памяти один раз. Промежуточные копии массивов минимизированы: модули `core/` работают с видами (views) NumPy-массивов, а не с их копиями, везде где это безопасно.

Максимальный размер файла, поддерживаемый без предупреждения: 100 МБ (≈ 12 млн строк при 8 байтах на значение и 8 колонках). При превышении этого порога выводится предупреждение с рекомендацией использовать формат Parquet.

Приложение не хранит историю всех сеансов в памяти. При запуске нового анализа объект предыдущего `AnalysisResult` освобождается сборщиком мусора Python, за исключением случая, когда пользователь не сохранил отчёт — тогда предыдущий результат сохраняется до явного закрытия или перезаписи.

## Работа с большими файлами

### Рекомендации для пользователя

При работе с записями длиннее 10 минут (600 секунд при 1000 Гц = 600 000 строк) рекомендуется:

- конвертировать CSV в Parquet с помощью встроенного инструмента экспорта (кнопка «Конвертировать в Parquet» на странице импорта);
- при необходимости анализировать отдельные сегменты записи через выбор временного диапазона.

### Адаптация алгоритмов

Алгоритм поиска пиков (алгоритм 6) при больших массивах работает с децимированным спектром: если количество частотных точек превышает 50 000, PSD вычисляется с более длинными сегментами, уменьшая разрешение, но сохраняя скорость отклика.

Скользящий RMS для записей длиннее 10 минут вычисляется с адаптивным шагом, уменьшая количество точек в тренде до 10 000 максимум.

## Отрисовка графиков

Построение графиков больших массивов данных выполняется с децимацией отображаемых точек. При наличии более 10 000 точек на графике сигнала данные децимируются до 5 000 точек для отображения — без изменения данных для расчётов.

Метод децимации: выбор точек через равный шаг (не усреднение). Это сохраняет пики и резкие переходы.

При изменении масштаба графика (зум) выполняется перерасчёт децимации для видимой области, что обеспечивает детализацию при приближении.
