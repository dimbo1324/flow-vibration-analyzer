# 07. Пользовательские сценарии — Industrial Vibration Analyzer

## Назначение документа

Этот документ описывает типичные сценарии работы с приложением от лица конкретных пользователей. Каждый сценарий показывает, с какой задачей приходит пользователь, какие шаги он выполняет и какой результат получает. Сценарии используются при проектировании интерфейса и при написании тестов.

---

## Персонажи

### Михаил — инженер-расчётчик теплообменного оборудования

Михаил работает в конструкторском бюро. Его задача — проверить, не грозит ли новой модели теплообменника вибрационное разрушение труб при рабочих условиях. У него есть данные измерений с испытательного стенда и результаты CFD-расчёта. Михаил хорошо знает физику задачи, но не является программистом.

### Анна — исследователь, аспирантка

Анна ведёт экспериментальное исследование обтекания тандема цилиндров. Её задача — обработать данные PIV-измерений, построить спектры вибраций при разных скоростях потока и сравнить с расчётами из диссертации. Анна умеет работать с Python, но ценит готовый инструмент, не требующий написания скриптов для каждого нового файла.

### Игорь — старший инженер на производстве

Игорь отвечает за вибрационный мониторинг действующего теплообменника. Периодически он получает файлы с записями виброускорений от эксплуатационного персонала и должен быстро оценить, есть ли признаки опасного режима. Ему нужен быстрый результат — не более пяти минут от загрузки до заключения.

## Сценарий 1 — Базовый анализ вибраций с формированием отчёта

**Пользователь:** Михаил  
**Цель:** Оценить спектральные характеристики вибраций трубы и получить заключение о риске резонанса для включения в технический отчёт.

**Шаги:**

1. Михаил открывает приложение и видит страницу «Сводка».
2. Нажимает кнопку «Импорт» и выбирает файл `stend_run_047.csv`.
3. Приложение показывает сводку по файлу: 60 секунд записи, 60 000 строк, частота дискретизации 1000 Гц, обнаружены 2 выброса.
4. Михаил назначает колонки на роли: `time` → временна́я ось, `acc_y` → виброускорение Y.
5. Переходит на вкладку «Физика» и вводит параметры: диаметр трубы 18 мм, скорость потока 1,8 м/с, плотность воды 998 кг/м³, вязкость 1,0×10⁻³ Па·с, собственная частота трубы 85 Гц.
6. Нажимает «Запустить анализ».
7. Через 4 секунды видит на странице «Сводка»: доминирующий пик 32,1 Гц, расчётная частота срыва вихрей 31,4 Гц, уровень риска — «Наблюдение».
8. Переходит на вкладку «Спектр» и изучает PSD-график. Видит чёткий пик на 32,1 Гц и его вторую гармонику на 64,2 Гц.
9. Нажимает «Экспорт отчёта» и сохраняет PDF.
10. Открывает отчёт и убеждается, что в нём есть все необходимые данные для включения в технический документ.

**Результат:** Инженер получил оформленный PDF-отчёт с результатами анализа, физическими коэффициентами и рекомендацией за 7 минут работы с приложением.

## Сценарий 2 — Сравнение эксперимента с CFD-расчётом

**Пользователь:** Анна  
**Цель:** Сопоставить экспериментальный профиль скорости за цилиндром с профилем из расчёта ANSYS CFX и количественно оценить точность модели.

**Шаги:**

1. Анна загружает экспериментальный файл `piv_cross_section_x2d.csv` с профилем скорости Vx(y).
2. Назначает колонки: `y_coordinate` → координата, `vx_mean` → скорость.
3. Загружает второй файл `cfd_profile_x2d.csv` с расчётным профилем.
4. Переходит на вкладку «Профили».
5. Видит совмещённый график двух профилей. Экспериментальный и расчётный профили близки в ядре потока, но расходятся в пристенной области.
6. В таблице метрик видит: RMSE = 0,043 м/с, MAPE = 4,7%, коэффициент корреляции r = 0,987.
7. Анна сохраняет файл `validation_results.csv` для дальнейшей обработки в Python.

**Результат:** Получена количественная оценка качества CFD-модели. Данные экспортированы для последующего анализа.

## Сценарий 3 — Быстрая оценка в полевых условиях

**Пользователь:** Игорь  
**Цель:** Быстро определить, есть ли в записи виброускорений признаки опасного режима.

**Шаги:**

1. Игорь получает файл `he_unit_3_acc_20250610.csv` от эксплуатационного персонала.
2. Открывает приложение, перетаскивает файл в окно импорта.
3. Приложение автоматически определяет структуру файла и предлагает назначения колонок.
4. Игорь принимает автоматические назначения и нажимает «Запустить анализ» без изменения настроек.
5. На странице «Сводка» видит: доминирующий пик 61,3 Гц, уровень риска — «Критический», текст рекомендации: «Немедленно проверьте рабочий режим оборудования».
6. Игорь сохраняет отчёт и немедленно передаёт его руководителю.

**Результат:** Потенциально опасный режим обнаружен за 3 минуты без ручной настройки параметров.

## Сценарий 4 — Анализ нескольких режимов работы

**Пользователь:** Анна  
**Цель:** Построить серию спектров при разных скоростях потока и проследить смещение частоты срыва вихрей.

**Шаги:**

1. Анна последовательно загружает 6 файлов с измерениями при скоростях 0,5; 1,0; 1,5; 2,0; 2,5 и 3,0 м/с.
2. Для каждого файла запускает анализ и записывает найденный доминирующий пик.
3. Экспортирует из каждого сеанса файл `physics_summary.csv`.
4. Объединяет результаты в Excel для построения зависимости fs(V).

**Замечание для будущей версии:** В текущей версии приложение работает с одним сеансом за раз. Пакетная обработка нескольких файлов — задача дорожной карты (см. `20_roadmap.md`).

## Сценарий 5 — Работа с некачественными данными

**Пользователь:** Михаил  
**Цель:** Проанализировать запись, в которой датчик временно терял сигнал.

**Шаги:**

1. Михаил загружает файл с 8% пропущенных значений в колонке давления.
2. Приложение показывает предупреждение: «Обнаружено 8,1% пропусков. Линейная интерполяция применяется только к промежуткам длиной до 50 мс. Промежутки длиннее 50 мс заменены нулями. Качество спектрального анализа может быть снижено».
3. Михаил видит, где именно находятся пропуски, на графике временно́го сигнала (отмечены серым фоном).
4. Он принимает решение ограничить анализ участком записи без пропусков, выбирая временно́й диапазон через форму.
5. Анализ выполняется по выбранному участку.

**Результат:** Приложение не скрывает проблему с данными, а помогает инженеру принять осознанное решение о способе её обработки.

## Сценарий 6 — Установка приложения

**Пользователь:** Любой  
**Цель:** Установить приложение на новой рабочей машине.

**Шаги:**

1. Пользователь запускает `IVA_Setup_1.0.0.exe`.
2. Установщик предлагает путь установки (`C:\Program Files\IVA\`) и возможность изменить его.
3. Создаётся ярлык на рабочем столе.
4. После установки приложение запускается сразу.
5. При первом запуске создаётся папка для результатов в `Документах`.

Установка не требует прав администратора. Python и другие компоненты не требуется устанавливать отдельно.
