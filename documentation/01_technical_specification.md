# 01. Техническое задание — Industrial Vibration Analyzer

## Назначение документа

Этот документ описывает полный набор требований к приложению: что оно должно уметь делать, как оно должно себя вести, при каких условиях работать и какие ограничения соблюдать. Документ является основным ориентиром при разработке, тестировании и приёмке результатов.

---

## Функциональные требования

### Модуль импорта данных

Приложение должно принимать файлы в форматах CSV, Parquet и Excel (`.xlsx`). При загрузке файла приложение автоматически определяет разделитель, кодировку, структуру колонок и частоту дискретизации. Если частота дискретизации явно не указана в файле, пользователь вводит её вручную.

Приложение должно отображать сводку по загруженным данным: количество строк, количество колонок, временной диапазон, обнаруженные пропуски, выбросы и нарушения монотонности временной оси.

Приложение должно позволять пользователю выбрать, какие колонки использовать для анализа, и присвоить им смысловые роли: время, виброускорение, давление, скорость потока.

### Модуль предобработки сигнала

Приложение должно выполнять следующие операции над выбранным сигналом:

- устранение постоянной составляющей (вычитание среднего значения);
- удаление одиночных выбросов методом скользящего медианного порога с настраиваемым порогом чувствительности;
- линейная интерполяция коротких пропусков (не более 0,5% от длины записи);
- полосовой фильтр Баттерворта с настраиваемыми граничными частотами;
- нормирование сигнала к физическим единицам (м/с², Па, м/с) на основании коэффициента преобразования датчика.

Каждая операция предобработки должна включаться и отключаться независимо. Порядок применения операций фиксирован и описан в документе `09_processing_pipeline.md`.

### Модуль спектрального анализа

Приложение должно вычислять спектральную плотность мощности методом Уэлча с настраиваемыми параметрами: тип окна (Ханнинга, Хэмминга, прямоугольное), длина сегмента, перекрытие сегментов. Шаг по частоте не должен превышать 0,5 Гц при стандартных параметрах.

Приложение должно автоматически находить пики в спектре, вычислять их частоту, амплитуду и ширину на уровне половинной мощности. Для каждого найденного пика должна выводиться интерпретация: срыв вихрей, гармоника, собственная частота конструкции или неопределённый источник.

Приложение должно вычислять RMS-значение сигнала в заданной частотной полосе и в скользящем временном окне.

### Модуль физических расчётов

Приложение должно вычислять следующие безразмерные и размерные характеристики течения:

- число Рейнольдса по диаметру цилиндра и средней скорости потока;
- число Струхаля для одиночного цилиндра и для тандема цилиндров;
- расчётную частоту срыха вихрей на основании числа Струхаля, скорости потока и диаметра цилиндра;
- приведённую скорость (velocity ratio), характеризующую близость рабочего режима к зоне lock-in.

Все параметры течения и геометрии пользователь вводит через форму с проверкой допустимых диапазонов.

### Модуль сравнения с расчётными данными

Приложение должно позволять загрузить второй файл с расчётными (CFD) данными и сопоставить их с экспериментальными. Для сопоставления вычисляются:

- среднеквадратичное отклонение (RMSE);
- средняя абсолютная погрешность (MAE);
- средняя абсолютная процентная погрешность (MAPE);
- коэффициент корреляции Пирсона.

Результат сравнения отображается в виде совмещённых графиков и таблицы расхождений.

### Модуль оценки риска резонанса

Приложение должно классифицировать текущий режим работы по трём уровням риска: безопасный, наблюдение, критический. Классификация основана на отношении расчётной частоты срыва вихрей к собственной частоте конструкции и на амплитуде пика в спектре вибраций.

Для каждого уровня риска приложение выводит текстовую рекомендацию для инженера.

### Модуль отчётности

Приложение должно генерировать отчёт в формате PDF и CSV. Отчёт содержит: параметры анализа, ключевые результаты, основные графики, таблицу найденных частотных пиков, физические коэффициенты и итоговую рекомендацию.

## Нефункциональные требования

### Производительность

Обработка файла объёмом до 50 МБ должна завершаться не более чем за 10 секунд на целевой конфигурации оборудования. Интерфейс не должен блокироваться во время вычислений: расчёт выполняется в фоновом потоке, пользователь видит индикатор прогресса.

Построение всех графиков на странице должно занимать не более 3 секунд после завершения расчёта.

### Надёжность

Приложение не должно аварийно завершаться при получении некорректных входных данных. Любая ошибка, возникающая внутри расчётного модуля, должна перехватываться, записываться в журнал и отображаться пользователю в виде понятного сообщения без технических деталей реализации.

Результаты расчёта не должны изменяться между запусками при одинаковых входных данных и настройках. Детерминированность является обязательным требованием.

### Удобство использования

Новый пользователь, знакомый с задачами вибродиагностики, должен выполнить полный цикл анализа — от загрузки файла до получения отчёта — не более чем за 15 минут без изучения документации.

Все элементы интерфейса должны иметь подсказки. Ключевые физические параметры сопровождаются краткими пояснениями прямо в интерфейсе.

### Совместимость

Приложение должно работать на Windows 11 (64-bit) при наличии минимум 4 ГБ оперативной памяти и 500 МБ свободного места на диске. Установка не требует прав администратора для пользовательского профиля.

### Безопасность

Приложение работает полностью локально и не передаёт данные во внешние сети. Все входные файлы проходят проверку структуры и типов данных перед обработкой. Некорректные файлы отвергаются с сообщением об ошибке.

### Поддерживаемость

Код приложения должен быть покрыт автоматическими тестами не менее чем на 80% для расчётных модулей. Добавление нового типа анализа не должно требовать изменений в модулях интерфейса или импорта данных.

## Ограничения и допущения

Приложение не поддерживает работу в режиме реального времени — анализ ведётся только по заранее записанным данным. Поддержка потоковой передачи данных с оборудования является задачей будущих версий.

Приложение не поддерживает работу с трёхмерными векторными полями в текущей версии. Анализ профилей скорости ограничен одномерным и двумерным случаями.

Количество одновременно открытых файлов ограничено двумя: экспериментальный и расчётный. Поддержка сравнения нескольких наборов данных является задачей будущих версий.
