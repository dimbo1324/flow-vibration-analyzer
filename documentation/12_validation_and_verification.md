# 12. Верификация и валидация — Industrial Vibration Analyzer

## Назначение документа

Этот документ описывает, как проверяется корректность расчётных модулей приложения. Верификация отвечает на вопрос «правильно ли реализован алгоритм?», валидация — «правильный ли алгоритм выбран для данной задачи?».

---

## Различие верификации и валидации в контексте проекта

**Верификация** — проверка того, что код делает именно то, что написано в спецификации. Например: функция расчёта числа Рейнольдса возвращает правильное значение при известных входных данных.

**Валидация** — проверка того, что результаты приложения согласуются с физической реальностью. Например: спектральный анализ синтетического сигнала с известной частотой даёт пик именно на этой частоте с допустимой погрешностью.

Оба типа проверки необходимы и дополняют друг друга.

---

## Верификационные тесты расчётных модулей

### Расчёт числа Рейнольдса

Аналитическое решение: `Re = V * D / ν = (2.0 * 0.012) / (1.004e-6) ≈ 23 904`.

Тест:
```python
result = reynolds_calculator.calculate(V=2.0, D=0.012, rho=998.0, mu=1.002e-3)
assert abs(result - 23904) / 23904 < 0.001  # погрешность < 0.1%
```

### Расчёт частоты срыва вихрей

При St = 0.21, V = 2.0 м/с, D = 0.012 м: `fs = St * V / D = 35.0 Гц`.

Тест:
```python
result = vortex_frequency.calculate(St=0.21, V=2.0, D=0.012)
assert abs(result - 35.0) < 0.01
```

### Устранение постоянной составляющей

Тест: сигнал с заданным смещением 5.0 после обработки должен иметь среднее значение ≈ 0.

```python
signal = np.sin(np.linspace(0, 10, 1000)) + 5.0
result = preprocessor.remove_mean(signal)
assert abs(np.mean(result)) < 1e-10
```

### Полосовой фильтр — подавление вне полосы

Тест: синусоида на частоте 500 Гц при полосе пропускания 5–250 Гц должна быть подавлена до уровня менее 1% амплитуды.

```python
t = np.linspace(0, 1, 1000)
signal_high_freq = np.sin(2 * np.pi * 500 * t)
result = filter_module.apply_bandpass_filter(signal_high_freq, 1000, 5, 250, 4)
assert np.max(np.abs(result)) < 0.01
```

### Вычисление RMS

Аналитически: RMS синусоиды амплитуды A равен `A / sqrt(2)`. При A = 2.0: RMS = 1.4142...

```python
t = np.linspace(0, 10, 10000)
signal = 2.0 * np.sin(2 * np.pi * 50 * t)
result = rms_calculator.calculate_total_rms(signal)
assert abs(result - 2.0 / np.sqrt(2)) < 0.001
```

### Метод Уэлча — обнаружение известного пика

Синтетический сигнал: синусоида 40 Гц + гауссовый шум. Ожидается пик в диапазоне 39.5–40.5 Гц.

```python
t = np.linspace(0, 60, 60000)
signal = 1.0 * np.sin(2 * np.pi * 40 * t) + 0.1 * np.random.randn(60000)
freqs, psd = psd_calculator.calculate_psd(signal, 1000, default_settings)
peak_freq = freqs[np.argmax(psd)]
assert 39.5 <= peak_freq <= 40.5
```

### Метрики сравнения

Тест на идентичных массивах: RMSE = 0, MAE = 0, r = 1.

Тест на массивах со смещением 1.0: RMSE = MAE = 1.0, r = 1.0.

Тест с нулями в знаменателе MAPE: функция возвращает `None`, а не исключение.

## Валидационные тесты на реальных данных

### Эталонный синтетический датасет

В `tests/fixtures/` хранятся синтетические файлы, сгенерированные скриптом `scripts/generate_synthetic_data.py` с заранее известными параметрами:

| Параметр | Значение |
|---|---|
| Частота дискретизации | 1000 Гц |
| Длительность записи | 60 с |
| Частота срыва вихрей | 40.0 Гц |
| Амплитуда пика | 1.0 (усл. ед.) |
| Уровень шума | 0.15 (SNR ≈ 16 дБ) |
| Наличие выбросов | 5 случайных выбросов |
| Пропуски | 0.5% |

Полный пайплайн на этом датасете должен выдавать доминирующий пик в диапазоне 39.0–41.0 Гц.

### Валидация по данным диссертации

В `tests/fixtures/dissertation/` хранится анонимизированный экспериментальный датасет из диссертационного исследования (показания с акселерометра при скорости потока 1.8 м/с, цилиндр D = 12 мм, вода).

Ожидаемые результаты (подтверждены в диссертации):
- доминирующий пик: 31 ± 2 Гц;
- Re ≈ 21 500 (турбулентный режим);
- St ≈ 0.21 ± 0.02.

Тест считается пройденным, если все три условия выполняются.

### Валидация метрик сравнения на аналитическом профиле

Использован аналитический профиль скорости в канале (параболический профиль Пуазейля). Расчётный профиль задаётся той же формулой с добавлением 2% случайного шума. Ожидаемое MAPE < 3%.

## Допуски на погрешность вычислений

| Расчётная величина | Допустимая погрешность | Обоснование |
|---|---|---|
| Число Рейнольдса | < 0.1% | Прямая формула, нет итераций |
| Число Струхаля | < 2% | Табличная интерполяция |
| Частота срыва вихрей | < 2% | Производная от St и скорости |
| Частота доминирующего пика | < delta_f | Ограничено разрешением по частоте |
| RMS синусоиды | < 0.1% | Аналитическое решение |
| RMSE и MAE | < 0.01% | Прямые формулы |

## Регрессионное тестирование

Все верификационные тесты выполняются автоматически при каждом запуске CI (см. `13_testing_strategy.md`). Если результат тестового расчёта изменился, тест падает даже при незначительном отклонении. Это предотвращает случайное изменение поведения алгоритмов при рефакторинге.

Базовые значения (golden values) хранятся в `tests/fixtures/expected_results.json` и обновляются вручную при намеренном изменении алгоритма.
