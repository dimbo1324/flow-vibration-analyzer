# 22. Кейс для портфолио — Industrial Vibration Analyzer

## Назначение документа

Этот документ описывает проект IVA как самостоятельный кейс: контекст задачи, принятые решения, технические достижения и демонстрируемые компетенции. Он предназначен для включения в портфолио разработчика и для презентации проекта потенциальным работодателям или коллаборантам.

---

## Короткое описание проекта

**Industrial Vibration Analyzer (IVA)** — инженерное настольное приложение для Windows 11, решающее задачу анализа потоково-индуцированных вибраций в теплообменном оборудовании. Пользователь загружает данные с датчиков вибраций или давления, приложение автоматически вычисляет спектральные характеристики, физические коэффициенты течения и оценивает риск попадания конструкции в резонансный режим.

Проект вырос из реального исследования — магистерской диссертации по гидродинамике теплообменников. Это не учебный пример, а инструмент, решающий конкретную инженерную задачу.

---

## Контекст и проблема

В ходе диссертационного исследования накапливался большой объём экспериментальных данных: записи виброускорений, профили скорости из PIV/LDV измерений, спектры пульсаций давления. Каждый новый набор данных обрабатывался заново — через разрозненные Python-скрипты, которые не имели общего интерфейса, не сохраняли параметры анализа и не формировали воспроизводимых отчётов.

Инженерная задача была сформулирована так: нужен инструмент, который принимает данные измерений, выполняет стандартную цепочку обработки и выдаёт заключение о безопасности текущего режима работы оборудования — без необходимости каждый раз писать скрипт с нуля.

Параллельно стала очевидна ценность такого инструмента за пределами одной диссертации: аналогичные задачи возникают при анализе трубных пучков, паропроизводящих каналов и других теплообменных элементов, где поток теплоносителя способен возбуждать опасные колебания конструкции.

## Архитектурные решения

### Четырёхслойная архитектура

Приложение разделено на строго изолированные слои: `ui` → `app` → `core` → `infrastructure`. Зависимости направлены только сверху вниз. Это не академическое упражнение — решение принято по конкретной причине: научное приложение склонно смешивать расчётную логику с интерфейсной, что делает алгоритмы непроверяемыми и хрупкими при изменениях.

Результат: все расчётные модули в `core/` тестируются независимо от GUI. Их можно запускать в командной строке или использовать как библиотеку в Jupyter Notebook.

### Детерминированный конвейер

Порядок шагов обработки фиксирован и задокументирован. Одни и те же входные данные всегда дают один и тот же результат. Для научного инструмента это не опция — это требование воспроизводимости исследования.

### Неизменяемые модели данных

Все внутренние структуры данных (`@dataclass(frozen=True)`) не могут быть изменены после создания. Это устраняет целый класс ошибок, когда один модуль неожиданно изменяет данные, используемые другим.

### Фоновое выполнение через QThread

Расчёт выполняется в отдельном потоке. Интерфейс остаётся отзывчивым. Пользователь видит прогресс в реальном времени и может отменить операцию.

## Что было реализовано

- Полный анализ сигнала: предобработка (фильтр Баттерворта, удаление выбросов методом скользящего медианного порога, заполнение пропусков), спектральный анализ методом Уэлча, поиск и интерпретация пиков
- Расчёт физических коэффициентов: Re, St, частота срыва вихрей, приведённая скорость
- Трёхуровневая оценка риска резонанса на основе close proximity к собственной частоте конструкции
- Сравнение экспериментальных данных с CFD-расчётами (RMSE, MAE, MAPE, коэффициент корреляции Пирсона)
- Генерация PDF-отчёта с графиками, таблицами и итоговым заключением
- Поддержка форматов CSV, Parquet, Excel с автоматическим определением структуры
- Семистраничный инженерный интерфейс с тёмной темой и адаптивной компоновкой
- Установщик `.exe` для Windows 11 без зависимости от установленного Python
- 23 документа: от технического задания до глоссария и дорожной карты

## Нетривиальные технические решения

**Устойчивая оценка выбросов через MAD.** Стандартное отклонение не подходит для обнаружения выбросов, потому что оно само смещается под влиянием выбросов. Применена оценка через медианное абсолютное отклонение (MAD) с масштабирующим коэффициентом 1.4826 для согласованности с нормальным распределением. Это стандартная практика в надёжной статистике, реализованная аккуратно с документированным обоснованием.

**Filtfilt вместо стандартного IIR-фильтра.** Обычный фильтр (lfilter) вносит фазовый сдвиг — события во времени сдвигаются после фильтрации. Для анализа пульсаций давления и вибраций это неприемлемо. Двойная фильтрация (filtfilt) устраняет фазовый сдвиг за счёт двукратного прохода. Решение зафиксировано в коде с комментарием о причине.

**Зависимость числа Струхаля от Re и конфигурации тандема.** Для одиночного цилиндра используется кусочно-линейная аппроксимация по таблице Бlevins. Для тандема — двумерная интерполяция по сетке (Re, L/D), построенной по экспериментальным данным диссертации. Это придаёт приложению уникальную ценность: числа Струхаля для конкретной исследованной конфигурации используются вместо общих справочных значений.

## Демонстрируемые компетенции

**Инженерная разработка программного обеспечения.** Проектирование архитектуры, разделение ответственности, управление зависимостями, написание тестов, автоматизация сборки и релиза.

**Обработка сигналов и цифровая фильтрация.** Практическое применение методов DSP: спектральный анализ, оконные функции, фильтрация Баттерворта, оценка RMS, обнаружение выбросов.

**Прикладная гидродинамика и вибродиагностика.** Понимание физики потоково-индуцированных вибраций, применение безразмерных критериев подобия, интерпретация спектральных данных в контексте реальной инженерной задачи.

**Документирование технических решений.** Способность описать архитектуру, алгоритмы, сценарии пользователей и требования к системе так, чтобы новый разработчик мог быстро войти в проект.

**Production-ready подход.** Обработка ошибок, журналирование, проверка входных данных, детерминированность расчётов, автоматические тесты, чеклист релиза.
